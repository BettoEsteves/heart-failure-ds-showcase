# Support Guide

## Purpose
Define how users and contributors should request help.

## Where to Ask
- Usage questions: open a discussion or issue with context.
- Bug reports: open an issue with steps to reproduce.
- Security concerns: follow `SECURITY.md` private reporting flow.

## Recommended Issue Content
- Objective and expected behavior
- Current behavior
- Environment details
- Logs or evidence (without secrets)
- Proposed workaround (if any)

## Response Expectations
- Triage should classify impact and priority.
- Work should be tracked in `docs/BoardTasks.md`.
